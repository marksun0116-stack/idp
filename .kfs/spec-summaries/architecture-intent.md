# Architecture intent (agent summary)

**Full spec:** `.kfs/specs/architecture_intent_spec.md`  
**Path:** `docs/architecture/*_architecture_intent.md` (team-specific names)

## Required content

- System context and scope
- Quality attributes (NFRs) — link enforceable ones to **CON-*** / **INV-***
- Boundaries, deployment, data ownership, integrations
- Security / tenancy posture
- Open decisions → **DEC-*** or explicit questions

## Before first FEAT (RULE-013)

At least one architecture intent doc in **`docs/architecture/`**.

## Do not

- Put enforceable API/security rules only in prose — use **CONR-*** / **CON-***
